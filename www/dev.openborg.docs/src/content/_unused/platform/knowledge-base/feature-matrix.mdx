---
title: Feature Matrix
description: Code-backed capability map by subsystem, with maturity notes.
---

## Runtime and ingress

Implemented:

- session-first ingress resolution
- actor binding precedence and persistence
- HTTP text/audio ingress
- managed Telegram/Discord connector supervision

Maturity notes:

- unknown connector providers are config-only (not started)
- connector feature parity is not identical across Telegram and Discord

## Providers and models

Implemented:

- provider CRUD
- provider kind validation
- runtime ordering via preferred-provider settings
- model discovery endpoint per provider

Maturity notes:

- unknown provider kinds are ignored by active resolver
- inferred model defaults are not auto-persisted

## Apps and integrations

Implemented:

- built-in app reconciliation on init/start
- custom app/capability/connection/secret CRUD
- OAuth start/callback flow with PKCE + state
- app secret projection into CodeMode env

Maturity notes:

- capability rows are metadata unless matching runtime tool execution path exists

## Memory graph

Implemented:

- fact state/retract/list
- entity create/get/search/explore
- schema definition tools
- search + explorer HTTP reads

Maturity notes:

- projection is asynchronous/eventual
- search behavior is entity-first and may need follow-up reads

## TaskGraph

Implemented:

- durable task records with DAG edges
- review workflow
- comments/events timelines
- queue helpers (`nextTask`, `reconcileInProgress`)

Maturity notes:

- REST is subset of fuller tool surface
- review-gated transitions are intentionally strict

## Clockwork

Implemented:

- job CRUD and status control
- once/cron schedule record handling
- due-job polling scaffold loop

Maturity notes:

- end-to-end autonomous dispatch path is still maturing

## Execution engines

Implemented:

- CodeMode JS runtime with Borg SDK context
- ShellMode host command execution

Maturity notes:

- CodeMode execution is globally serialized
- ShellMode timeout setting is present but currently not hard-enforced by process cancellation path

## API surface

Implemented:

- broad REST control plane + ingress
- standalone GraphQL crate with typed schema

Maturity notes:

- `/graphql` is not mounted in active `borg-api` router yet
