---
title: Known Caveats
description: Important constraints and edge cases to account for in docs and production usage.
---

## GraphQL transport not mounted

- `crates/borg-gql` exists with broad schema coverage.
- Active `borg-api` router does not currently expose `/graphql`.
- Runtime-wrapper GraphQL mutations include placeholder error behavior in standalone mode.

## Provider/runtime mismatch risks

- Provider records with unsupported `provider_kind` persist but are ignored by active resolver.
- Discovery-inferred defaults are suggestions only unless persisted.

## Port and connector constraints

- Only Telegram and Discord are managed connector providers currently.
- Unknown connector providers are skipped at startup.
- Connector ingress is text-first; audio ingress is HTTP route today.
- Private connectors with empty allow-lists deny all external users.

## TaskGraph surface asymmetry

- REST is intentionally smaller than full `TaskGraph-*` tool surface.
- Review-related status transitions are restricted; direct shortcuts are rejected.

## Clockwork maturity

- Job storage and controls are present.
- Scheduler loop currently emphasizes due-job polling/scaffold logging; full dispatch semantics are still evolving.

## Capability dispatch assumptions

- App capability `mode` is metadata in current runtime behavior.
- Execution depends on available toolchain implementation, not mode label alone.

## ShellMode timeout caveat

- ShellMode context includes timeout values.
- Current engine does not hard-kill process execution by timeout in active path.

## Documentation hygiene caveat

Because this codebase is moving quickly, docs should always call out date-sensitive maturity notes explicitly when behavior is not yet production-hardened.
