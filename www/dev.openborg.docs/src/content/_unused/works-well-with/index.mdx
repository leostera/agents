## Purpose of this section

These pages are for **complementary** tools, not direct replacements.

Each page answers five practical questions:

1. What each product should own.
2. Where handoff happens.
3. How to implement the first combined workflow.
4. What to measure to know the pairing works.
5. What anti-patterns to avoid.

## How to read these pages

- If you are evaluating alternatives, use [Comparisons](/compare/).
- If you already use one of these tools and want openborg in the stack, start here.

## Fast adoption rule

For any pairing, do this first:

1. Keep the existing tool as-is for what it already does well.
2. Move only one repeatable, high-cost workflow into openborg.
3. Add explicit handoff checkpoints before expanding scope.
