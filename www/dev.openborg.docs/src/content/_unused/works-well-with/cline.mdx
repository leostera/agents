---
title: openborg works well with Cline
description: Use Cline for transparent coding-agent iteration and openborg for durable execution control.
---

## Core value split

- **Cline**: interactive coding-agent iteration in-editor.
- **openborg**: repeatable runtime execution and lifecycle governance.

## Best-fit signal

This pairing is strong when your team says:

- "Cline helps us discover good implementations, but ops is still ad-hoc."
- "We need durable execution records beyond editor sessions."

## Handoff protocol

1. Iterate implementation approach in Cline.
2. Extract stable execution flow into openborg TaskGraph.
3. Use openborg review gates for high-risk transitions.
4. Keep Cline focused on code-level iteration.

## First workflow to implement

Start with one high-repetition engineering workflow:

1. codify steps in openborg tasks
2. enforce review before terminal completion
3. track event/comment audit trail
4. compare variance of outcomes over 2-3 cycles

## Success metrics

- lower execution variance across engineers
- less repeated troubleshooting for same workflow class
- clearer "who did what when" auditability

## Anti-patterns

- Using Cline and openborg to own identical responsibilities.
- Migrating all coding workflows into openborg.
- Skipping per-stage acceptance criteria.

## When not to pair

If workflows are mostly exploratory and non-repeatable, keep Cline-only until patterns stabilize.
