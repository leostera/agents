## Before enabling connectors

Make sure HTTP ingress path is stable first:

- provider configured
- actor + behavior configured
- HTTP turn succeeds end-to-end

## Telegram quick setup

Set bot token via CLI helper:

```bash
cargo run -p borg-cli -- config set ports.telegram "<BOT_TOKEN>"
```

Then confirm port record is enabled and has expected privacy/settings via `/api/ports`.

## Discord setup shape

Configure a Discord port with provider `discord` and settings JSON including bot token, then verify in `/api/ports` listing.

## Conversation semantics

- Telegram conversation key: chat id
- Discord conversation key: channel id

Sessions remain stable per `port + conversation_key`.

## Access control

Private connectors enforce `allowed_external_user_ids`:

- Telegram: numeric IDs or usernames (`@handle`)
- Discord: numeric snowflake IDs only

Empty allow-list on private connector blocks all external senders.

## Verification checklist

1. Port appears enabled in `/api/ports`.
2. Inbound message creates/uses expected session.
3. Actor binding resolves as expected.
4. Reply appears in channel.
5. Context snapshot visible via port session context API.

## Current connector caveat

Connector ingress is text-first. Audio ingress currently uses HTTP audio route.
