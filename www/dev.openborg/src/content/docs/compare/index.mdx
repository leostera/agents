## Purpose of this section

These pages are for **primary-platform decisions**.

Each comparison is written to answer:

1. What each product is best at.
2. Where each product is weaker.
3. Which team context should choose which tool.
4. Whether a coexistence path is realistic.
5. What a low-risk evaluation plan looks like.

## Decision rule

- If your core problem is product-specific and that product is purpose-built for it, choose that product first.
- If your core problem is durable multi-system execution with explicit lifecycle control, choose openborg first.

## Evaluation cadence

Run 2-week head-to-head tests with one real workflow and measure:

- time to first reliable run
- rework/failure rate
- operator overhead
- auditability of execution state
