## HTTP API surface

Provider endpoints:

- `GET /api/providers`
- `GET /api/providers/:provider`
- `PUT /api/providers/:provider`
- `DELETE /api/providers/:provider`
- `GET /api/providers/:provider/models`
- `POST /api/providers/openai/device-code/start`

## Upsert payload shape

`PUT /api/providers/:provider` accepts:

- `provider_kind` (required on create if not inferable)
- `api_key`
- `base_url`
- `enabled`
- `default_text_model`
- `default_audio_model`

Validation depends on kind:

- OpenAI/OpenRouter: require `api_key`.
- LM Studio/Ollama: require `base_url`.

## Discovery endpoint behavior

`GET /api/providers/:provider/models`:

- returns discovered model IDs
- returns inferred `default_text_model` and `default_audio_model`
- returns `404` when provider is missing or disabled
- returns `502` on upstream provider/model endpoint failures

## CLI workflow

```bash
cargo run -p borg-cli -- providers list
cargo run -p borg-cli -- providers get openai
cargo run -p borg-cli -- providers upsert openai --provider-kind openai --api-key <KEY>
cargo run -p borg-cli -- providers set default openai
```

`providers set default` writes runtime settings keys used by selection order.

## Recommended configuration flow

1. Create provider record with valid kind + credentials/base URL.
2. Validate model discovery endpoint.
3. Persist explicit default text/audio models when you need deterministic behavior.
4. Set runtime default provider explicitly.
5. Re-verify by executing a simple turn and checking model/provider output.

## Common errors

- Missing required credentials/base URL => `400`.
- Unsupported `provider_kind` => `400`.
- Disabled provider queried for models => `404`.
- Upstream model endpoint failures => `502`.
